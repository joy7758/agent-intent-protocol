# AIP Terminology

## Intent Object

A machine-readable declaration of the requested outcome and any declared constraints.

## Action Object

A machine-readable declaration of a concrete operation proposed or executed by an actor.

## Result Object

A machine-readable declaration of the interaction outcome status and any references to outputs or evidence.

## Actor Reference

A stable reference to the acting identity surface. AIP treats `actor_ref` as an external identity pointer rather than an embedded identity format.

## Correlation ID

A stable identifier used to associate related intent, action, and result objects across a run.

## Governance Checkpoint

A pre-execution decision point where an intent or action can be allowed, blocked, degraded, or redirected.
